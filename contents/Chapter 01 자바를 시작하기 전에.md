<h1>Chapter 01 자바를 시작하기 전에</h1>

<h3>자바란?</h3>
자바는 1996년 1월에 공식적으로 발표한 객체지향 프로그래밍 언어이며, 추후에 함수평 프로그래밍 기능이 추가되었다.<br>
현재 대부분의 자바는 SE(Standard Edition)을 의미한다.<br>
JDK(Java Development Kit)에서 JDK 8은 람다식과 스트림 API가 추가되면서 함수형 프로그래밍이 가능해졌다.
JDK9에서 G1(Garbage First)이라는 이름의 가비지 컬렉터(Gabage Collector)가 새로 추가되어 자동으로 메모리를 관리해주는 기능이 추가되었다.
<br><br>
<h3>자바 언어의 특징</h3>
<b>1. 운영체제에 독립적이다.</b>
<ul>
  <li>자바 응용프로그램은 자바가상머신(JVM)하고만 통신하고 JVM이 자바 응용프로그램으로부터 전달받은 명령어를 해당 운영체제가 이해할 수 있도록 변환하여 전달한다.</li>
  <li>따라서 자바로 작성된 프로그램은 운영체제와 하드웨어에 관계없이 실행 가능하다.</li>
</ul>
<b>2. 객체지향 언어이자 함수형 언어이다.</b><br>
<b>3. 비교적 배우기 쉽다.</b><br>
<b>4. 자동 메모리 관리 (Garbage Collector)</b>
<ul>
  <li>자바로 작성된 프로그램이 실행되면 가비지 컬렉터(Garbage Collector)가 자동적으로 메모리를 관리해주기 때문에 프로그래머는 메모리를 따로 관리 하지 않아도 되어 프로그래밍에 집중할 수 있도록 도와준다.</li>
</ul>
<b>5. 네트워크와 분산처리를 지원한다.</b>
<ul>
  <li>다양한 네트워크 프로그래밍 라이브러리(Java API)를 통해 비교적 짧은 시간에 쉽게 개발을 할 수 있도록 지원한다.</li>
</ul>
<b>6. 멀티쓰레드를 지원한다.</b>
<ul>
  <li>자바의 멀티쓰레드 프로그램은 시스템과는 관계없이 구현 가능하며, 관련된 라이브러리(Java API)가 제공되므로 구현이 쉽다.</li>
  <li>여려 쓰레드에 대한 스케줄링을 자바 인터프리터가 담당하게 된다.</li>
</ul>
<b>7. 동적 로딩(Dynamic Loading)을 지원한다.</b>
<ul>
  <li>자바는 실행 시 모든 클래스가 롸딩되지 않고 필요한 시점에 클래스를 로딩하여 사용할 수 있다.</li>
</ul>
<br>
<h3>JVM(Java Virtual Machine)</h3>
자바로 작성된 애플리케이션은 모두 가상 컴퓨터(JVM)에서만 실행되기 때문에, 자바 애플리케이션이 실행되기 위해서는 반드시 JVM이 필요하다.
Java 애플리케이션은 JVM을 한번 더 거치고 하드웨어에 맞게 완전히 컴파일된 상태가 아니고 실행 시에 해석(interpret)되기 때문에 속도가 느리다는 단점이 있다.
그리고 OS와 하드웨어에 독립적이라 다른 OS에서도 프로그램의 변경없이 실행이 가능하다. 단, JVM은 OS에 종속적이기 때문에 해당 OS에서 실행 가능한 JVM이 필요하다.
<br><br>
<h3>자바프로그램의 실행과정</h3>
1. 프로그램의 실행에 필요한 클래스(*.class파일)를 로드한다.<br>
2. 클래스파일을 검사한다.(파일형식, 악성코드 체크)<br>
3. 지정된 클래스에서 main(Stringp[] args)를 호출한다.
