<h1>1. 변수(variable)</h1>
<h3>변수(variable)란?</h3>
프로그래밍언어에서의 변수(variable)란, 단 하나의 값을 저장할 수 있는 메모리 공간을 의미한다. 새로운 값을 저장하면 기존의 값은 사라진다.
<br><br>
<h3>변수의 선언과 초기화</h3>

```
int age;
```
위 간단한 코드에서 int는 변수타입이고 age는 변수이름이다. age라는 이름의 변수를 선언했다. '변수타입'은 변수의 '종류'를 지정하는 것이다. '변수이름'은 말 그대로 변수에 붙인 이름이다. 그 이름을 이용해서 변수에 값을 저장하고, 저장된 값을 불러올 수 있다.<br>
변수를 선언하면, 메모리의 빈 공간에 '변수타입'에 알맞은 크기의 저장 공간이 확보되고, 이 저장공간에 값을 읽고 쓰는 것은 '변수이름'을 통해서 가능하다.
<br><br>
변수를 선언한 이후에는 반드시 변수를 초기화(Initialization)해야 한다. 알 수 없는 값(garbage value)가 남아있을 수 있기 때문이다. 변수의 값을 저장할 때는 대입 연산자'='를 사용한다. 즉, 변수의 초기화란 변수를 사용하기 전에 처음으로 값을 저장하는 것이다.
<br><br>
<h3>변수의 명명규칙</h3>
프로그래밍에서 사용하는 모든 이름을 식별자(identifier)라고 하며, 다음과 같은 규칙을 지켜야 한다.<br><br>
<b>1. 대소문자가 구분되며 길이에 제한이 없다.</b><br>
<b>2. 예약어를 사용해서는 안 된다.</b><br>
<b>3. 숫자로 시작해서는 안 된다.</b><br>
<b>4. 특수문자는 '_'와 '$'만을 허용한다.</b><br><br><br>

<h1>2. 변수의 타입</h1>
값(data)의 종류(type)에 따라 값이 저장될 크기와 저장형식을 정의한 것이 자료형(data type)이다. 자료형에는 문자형(char), 정수형(byte, short, int, long), 실수형(float, double) 등이 있다.
<br><br>
<h3>기본형과 참조형</h3>
기본형 변수는 실제 값(data)을 저장하는 반명, 참조형 변수는 어떤 값이 저장되어 있는 주소(memory address)를 값으로 갖는다. 자바는 참조형 변수 간의 연산을 할 수 없으므로 연산에 사용되는 것은 모두 기본형 변수이다.<br>
참조형 변수를 선언할 때는 변수의 타입으로 클래스 이름을 사용하므로 클래스의 이름이 참조 변수의 타입이 된다. 새로운 클래스를 작성한다는 것은 새로운 참조형 변수를 추가하는 셈이다. 참조형 변수는 null 또는 객체의 주소 (4 byte)를 값으로 갖는다.
<br><br>
<h3>기본형(primitive type)</h3>

| 자료형 | 1byte | 2byte | 4byte | 8byte |
|---|---|---|---|---|
| 논리형 | boolean |  |  |  |
| 문자형 |  | char |  |  |
| 정수형 | byte | short | int | long |
| 실수형 |  |  | float | double |

boolean은 다른 기본형과의 연산이 불가능하다. boolean 제외 나머지 기본형은 서로 연산과 변환이 가능하다. int타입의 변수는 대략 10자리 수(약 20억)의 값을 저장할 수 있다. 7~9자리 수를 계산할 때는 넉넉하게 long타입으로 변수를 선언하는 것이 좋다. <br>
float과 double의 경우 float은 정밀도 7자리, double은 정밀도 15자리로 높은 정밀도가 필요한 경우에는 변수의 타입으로 double을 선택해야 한다.
<br><br>
<h3>상수와 리터럴(constant & literal)</h3>
'상수(constant)'는 변수와 달리 값을 한번 저장하면 다른 값으로 변경할 수 없다. 변수의 타입 앞에 키워드 'final'을 붙이면 된다. 보통 선언과 동시에 초기화한다. 
<br><br>
<h3>리터럴</h3>
프로그래밍에서 리터럴은 그 자체로 값을 의미하는 것을 의미한다.
<br><br>
<h3>리터럴의 타입과 접미사</h3>
정수형과 실수형에 여러 타입이 존재하므로, 접미사를 붙여서 타입을 구분한다. 정수형의 경우, long 타입 리터럴에 접미사 'l' 또는 'L'을 붙이고, 접미사가 없으면 int 타입의 리터럴이다. byte와 short의 리터럴은 별도로 존재하지 않으며, 변수에 값을 저장할 때는 int타입의 리터럴을 사용한다.
<br>
실수형에서는 float타입의 접미사 'f'또는 'F'를 붙이고, double 타입의 리터럴에는 접미사 'd'또는 'D'를 붙인다. 실수형에서는 double이 기본 자료형이라서 접미사 'd'는 생략 가능하다.
<br><br>
<h3>타입의 불일치</h3>
리터럴의 타입은 저장될 변수의 타입과 일치하는 것이 보통이지만, 타입이 달라도 저장범위가 넓은 타입에 좁은 타입의 값을 저장하는 것은 허용된다.<br>
그러나 리터럴의 값이 변수의 타입의 범위를 넘어서거나, 타입이 변수의 타입보다 저장범위가 넓으면 컴파일 에러가 발생한다.

```
long l = 123;  // 가능, int보다 long이 더 범위가 넓다.
float f = 3.14  // 에러, float보다 double의 범위가 넓다.
```
<h3>지역 변수의 타입 추론 - var</h3>
JDK 10부터 지역변수(local variable)의 경우, 변수를 선언할 때 타입 대신 'var'를 사용할 수 있게 되었다. 컴파일러가 값의 타입을 보고 변수의 타입을 추론 할 수 있는 것이다. 'var'를 사용하면 코드가 간결해지고, 변경에도 유리해진다.<br>
만일 변수를 선언할 때 값을 대입하지 않거나 null을 대입하면, 변수의 타입을 추론할 수 없기 때문에 에러가 발생한다.
<br><br>
<h3>문자 리터럴과 문자열 리터럴</h3>
'A'와 같이 작은 따옴표로 문자 하나를 감싼 것을 '문자 리터럴'이라고 한다. 두 문자 이상은 큰 따옴표로 감싸야 하며 '문자열 리터럴'이라고 한다.<br>
문자열 리터럴은 ""안에 아무런 문자도 넣지 않는 것을 허용하며, 빈 문자열(empty String)이라고 한다. 그러나 문자 리터럴은 반드시 ''안에 하나의 문자가 있어야 한다.<br>
덧셈 연산자(+)를 이용하여 문자열을 결합할 수 있는데, 피연산자 중 어느 한 쪽이 String이면 나머지 한 쪽을 먼저 String으로 변환한 다음 두 String을 결합한다. 따라서 기본형 타입의 값을 문자열로 변환할 때 빈 문자열("")을 더해주면 된다.
<br><br>
<h3>텍스트 블럭(text blocks)</h3>
JDK15부터 다중행 문자열 리터럴(multiline string literal)을 작성할 수 있으며, 여러 줄로 이루어진 문자열을 작성하기 편리하게 해준다.
